## Фронтенд и бэкенд разработка — Практические занятия 1–5

Этот репозиторий содержит выполненные практические задания по дисциплине **«Фронтенд и бэкенд разработка»** (4 семестр). Каждое задание реализовано в отдельной части проекта и включает как фронтенд, так и бэкенд‑разработку, работу с API, препроцессорами, Postman и Swagger.

---

## Практическое занятие №1 — CSS‑препроцессоры (SASS/LESS)

**Цель:** изучить переменные, вложенность, миксины и модульность в препроцессорах.

**Что сделано:**
- Создана карточка товара (название, описание, фото).
- Использованы:
  - **минимум 2 переменные** (цвета, размеры),
  - **1 миксин** (например, для кнопки или тени),
  - **вложенная структура селекторов**,
  - **SASS** как основной препроцессор.
- Скомпилирован итоговый CSS.

**Результат:** стилизованная карточка товара с использованием возможностей препроцессора.

---

## Практическое занятие №2 — Сервер на Node.js + Express

**Цель:** реализовать собственный REST API.

**Что сделано:**
- Создан сервер Express.
- Реализованы CRUD‑операции для сущности **товар**:
  - `GET /products`
  - `GET /products/:id`
  - `POST /products`
  - `PATCH /products/:id`
  - `DELETE /products/:id`
- Добавлен парсинг JSON (`express.json()`).
- Добавлено логирование запросов.
- Данные хранятся в массиве.

**Результат:** полностью рабочий API для товаров.

---

## Практическое занятие №3 — JSON и внешние API

**Цель:** научиться работать с JSON, тестировать API и взаимодействовать с внешними сервисами.

**Что сделано:**

### 1. Тестирование собственного API
В Postman выполнены запросы:
- `GET /products`
- `POST /products`
- `DELETE /products/:id`

Скриншоты собраны в файл `result.pdf`.

### 2. Работа с внешним API
Выбран сервис **OpenWeatherMap**, получен API‑ключ.  
Выполнено **5 запросов**:
- Погода по городу  
- Погода по координатам  
- Прогноз на 5 дней  
- Погода по ID города  
- Погода по нескольким городам  

Скриншоты также добавлены в отчёт.

**Результат:** освоена работа с JSON‑ответами и внешними API.

---

## Практическое занятие №4 — API + React

**Цель:** связать backend и frontend в единое приложение.

**Что сделано:**
- Создано React‑приложение.
- Реализованы компоненты:
  - список товаров,
  - карточка товара,
  - модальное окно создания/редактирования.
- Подключён backend через **axios**.
- Реализованы операции:
  - загрузка товаров,
  - создание,
  - редактирование,
  - удаление.
- Добавлена поддержка CORS.
- Использован SASS для стилизации.

**Результат:** полноценный мини‑интернет‑магазин с React‑интерфейсом и Express‑сервером.

---

## Практическое занятие №5 — Расширенный REST API + Swagger

**Цель:** документировать API с помощью Swagger.

**Что сделано:**
- Установлены `swagger-jsdoc` и `swagger-ui-express`.
- Добавлены JSDoc‑аннотации ко всем CRUD‑маршрутам.
- Описана схема товара (`Product`).
- Swagger UI доступен по адресу:

```
http://localhost:3000/api-docs
```

**Результат:** интерактивная документация API, полностью соответствующая коду.
